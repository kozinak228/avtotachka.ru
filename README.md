# 🚘 АвтоТачка — Платформа по продаже автомобилей из Китая

**АвтоТачка** — это современная, полнофункциональная платформа (CMS/Каталог) для презентации, сравнения и продажи автомобилей, поставляемых из Китая. Проект разработан с использованием нативного PHP (MVC-подобная архитектура), базы данных MySQL и передовых фронтенд-технологий (Tailwind CSS, AJAX), что обеспечивает высокую скорость работы, безопасность и премиальный пользовательский опыт.

## 1. Архитектура и структура проекта

Проект построен на базе **паттерна MVC (Model-View-Controller)**, адаптированного под нативный PHP (без использования тяжелых фреймворков вроде Laravel). Это позволяет гибко управлять логикой, сохраняя высокую производительность.

- **`/app/database/` (Models & DB)** — содержатся классы и методы для подключения к БД (PDO) и абстрактные функции CRUD-операций (`selectAll`, `selectOne`, `insert`, `update`, `delete`).
- **`/app/controllers/` (Controllers)** — логика обработки форм, валидация данных, обработка сессий, авторизация, загрузка файлов (`users.php`, `cars.php`, `commentaries.php`).
- **Корневые файлы `*.php` (Views)** — фронтенд-шаблоны страниц (`index.php`, `single.php`, `compare.php` и т.д.).
- **`/app/include/` (Components)** — переиспользуемые блоки интерфейса (header, footer, sidebar).
- **`/admin/`** — защищенная директория панели управления, доступная только пользователям с ролью Admin.
- **`/assets/`** — статические файлы (CSS, скрипты, шрифты, загруженные изображения авто и аватарки).

## 2. База данных (Схема)

Используется реляционная база данных **MySQL**. Взаимодействие осуществляется исключительно через защищенные подготовленные выражения (PDO Prepared Statements).

**Основные таблицы:**
*   `users` — хранит данные пользователей (логин, email, хеш пароля, роль, путь к аватару, био).
*   `cars` — основная таблица автомобилей (название, описание, цена, год, двигатель, пробег, ID бренда, главная картинка).
*   `brands` — справочник марок автомобилей.
*   `car_images` — таблица дополнительных фотографий (One-to-Many к таблице `cars`), используется для слайдеров и галерей.
*   `comments` — система отзывов (привязка к ID авто и ID пользователя, текст, статус публикации).
*   `comment_likes` — система уникальных лайков (привязка пользователя к комментарию во избежание накруток).
*   `favorites` — сохраненные автомобили пользователей (Список "Избранное").

## 3. Основной функционал (Frontend)

Пользовательская часть сайта спроектирована с акцентом на **премиальный UI/UX дизайн** уровня ведущих автосалонов.

*   **Динамический Каталог и Поиск:**
    *   Пагинация и вывод карточек автомобилей.
    *   **Живой поиск (Live AJAX Search)**: результаты появляются в выпадающем списке моментально при вводе текста, без перезагрузки страницы.
    *   Сложные фильтры по марке, кузову, цене и т.д.
*   **Детальная страница автомобиля (`single.php`):**
    *   Отображение всех характеристик.
    *   Интерактивный **Swiper.js слайдер** для просмотра дополнительных фотографий авто (поддержка свайпов на смартфонах).
*   **Кабинет пользователя (`profile.php`):**
    *   Загрузка профильного фото (с интерактивной обрезкой через Cropper.js).
    *   Настройки безопасности (смена пароля).
*   **Модуль "Избранное" и "Сравнение":**
    *   Возможность добавить машину в избранное (AJAX запрос, визуальный отклик). Отдельная страница для просмотра отложенных машин в виде стильных 3D карточек.
    *   Страница технического сравнения (Compare) — таблица для сопоставления выбранных авто по параметрам.
*   **Система независимых отзывов:**
    *   Возможность авторизованным пользователям оставлять комментарии.
    *   **Умные лайки** — каждый пользователь может лайкнуть один комментарий только один раз (AJAX + валидация в БД).
*   **Адаптивность и Тёмная тема (Dark Mode):**
    *   Сайт 100% Mobile-friendly.
    *   Автоматическое/ручное переключение светлой и тёмной тем через сессии (без мерцаний экрана).

## 4. Панель администратора (Backend)

Административная часть спроектирована для удобного управления контентом и модерации. Присутствует эксклюзивный дизайн в стиле "Матрицы" (анимированный canvas-фон, неоновые акценты).

*   **Управление парком авто (CRUD):** 
    *   Удобная форма добавления нового автомобиля с WYSIWYG редактором (CKEditor) для красивого оформления описаний.
    *   Множественная загрузка фото (Drag & Drop сортировка фотографий через Sortable.js).
    *   Изменение статуса публикации (Черновик / Опубликовано).
*   **Модерация пользователей и отзывов:**
    *   Изменение ролей пользователей (назначение администраторов).
    *   Управление комментариями: просмотр длинных текстов через Bootstrap Modals, пагинация комментариев, возможность скрыть спам или удалить отзыв.
*   **Справочники:**
    *   Управление списком марок (Брендов).

## 5. Безопасность и оптимизация

При разработке были учтены строгие стандарты веб-безопасности:
1.  **Защита от SQL-Injection:** Все запросы к БД отправляются исключительно через `PDO Prepared Statements`. Ни одна переменная пользователя не попадает в SQL-запрос напрямую.
2.  **Защита от XSS (Cross-Site Scripting):** Повсеместное использование `htmlspecialchars()` при выводе данных из базы на экран (в отзывах, именах, описаниях).
3.  **Безопасная авторизация:** 
    *   Пароли пользователей **не хранятся в открытом виде**. Применяется строгая криптографическая функция `password_hash()` (алгоритм BCRYPT).
    *   Реализована защита формы авторизации от брутфорс-атак (ограничение попыток входа с JS/Session таймерами).
4.  **Управление доступом (Middleware):** Проверка прав администратора перед каждым входом в директорию `/admin/`. Если у сессии нет прав — происходит мгновенный редирект на главную.

*   **Верстка (Стилизация):**

## 7. Установка и развертывание

### Требования к серверу:
- PHP 8.0 или выше (с включенным расширением PDO).
- MySQL 5.7+ или MariaDB.
- Локальный сервер (XAMPP / MAMP / OpenServer / Docker).

### Пошаговая инструкция:
1.  **Клонирование репозитория:**
    ```bash
    git clone https://github.com/kozinak228/new_china_CAR_site_DINAMIC_AND_PHP.git
    ```
2.  **Настройка базы данных:**
    *   Создайте базу данных в phpMyAdmin (например, `dinamic-site`).
    *   Импортируйте структуру БД из приложенного SQL файла (если есть).
3.  **Подключение к БД:**
    *   Откройте файл `app/database/connect.php`.
    *   Запишите актуальные доступы к вашей локальной базе (по умолчанию обычно `root` без пароля).
    ```php
    $host = 'localhost';
    $db_name = 'dinamic-site';
    $db_user = 'root';
    $db_pass = '';
    ```
4.  **Базовые пути:**
    *   Откройте файл `path.php` в корне проекта.
    *   Убедитесь, что константа `BASE_URL` указывает на правильный локальный путь к папке сайта (например, `http://localhost/china_cars/`).
5.  **Запуск:**
    *   Откройте локальный адрес сайта в браузере. Вы можете войти в админку, создав пользователя и программно присвоив ему роль `admin = 1` через phpMyAdmin.
